# 🚀 성능 벤치마크 및 QA (Benchmarks & QA)

Drilling RPG는 "브라우저 환경에서의 한계 극복"을 목표로 하며, 다음 지표를 기준으로 성능을 관리하고 검증합니다.

---

## 1. 핵심 성능 지표 (KPIs)

| 측정 항목 | 목표 수치 | 현재 상태 | 비고 |
| :--- | :--- | :--- | :--- |
| **엔티티 동시 처리** | 5,000+ 개 | 최적화 중 | SoA 도입으로 1만 개 목표 |
| **렌더링 주사율 (Main)** | 144Hz | 달성 | 트리플 버퍼링 보간 적용 |
| **시뮬레이션 주사율 (Worker)** | 60Hz | 달성 | 로직 부하와 독립적 실행 |
| **메인-워커 지연 (Latency)** | < 2ms | 달성 | Transferable objects 사용 |
| **초기 로딩 시간** | < 3s | 달성 | 자동 자산 아틀라스 생성 |

---

## 2. 성능 프로파일링 가이드

### **크롬 개발자 도구 활용**
1.  **Performance 탭**: CPU Profile을 통해 워커 스레드의 `gameLoop` 함수 실행 시간을 체크합니다. (목표: < 10ms per frame)
2.  **Memory 탭**: Heap Snapshot을 찍어 `ArrayBuffer`가 올바르게 재사용되고 있는지, 메모리 누수가 없는지 확인합니다.
3.  **Layers 탭**: PixiJS의 드로우 콜(Draw Call) 횟수를 확인합니다. (목표: 아틀라스 활용으로 50회 미만 유지)

---

## 3. QA 체크리스트

### **시각적 품질 (Visual)**
- [ ] 144Hz 이상의 고주사율 모니터에서 이동 시 잔상이나 지터(Jitter)가 없는가?
- [ ] 창 크기 조절(Resize) 시 배경과 UI 요소가 깨지지 않고 재배치되는가?
- [ ] 특수 효과(조명, 파티클)가 겹칠 때 프레임 드랍이 발생하는가?

### **게임 로직 (Logic)**
- [ ] 5분 이상의 장시간 플레이 시 메모리 사용량이 계속해서 증가하는가? (GC 체크)
- [ ] 고속 이동 시 타일 충돌 판정이 누락되어 벽을 뚫는 현상이 있는가?
- [ ] 다른 브라우저 탭으로 이동했다가 돌아왔을 때 게임 루프가 정상적으로 복구되는가?

---

## 4. 성능 최적화 팁 (Tips)

- **SoA 전환**: 현재 WIP(Work In Progress) 상태인 SoA 구조를 완성하여 CPU 캐시 효율을 극대화합니다.
- **오프스크린 렌더링**: 워커의 `OffscreenCanvas` 지연을 최소화하기 위해 메인 스레드의 DOM 조작을 최소화합니다.
- **에셋 지연 로드**: 필수적인 아틀라스 외의 사운드나 고해상도 자산은 게임 플레이 중에 백그라운드에서 로드합니다.
