# [자산 가이드] 자동화 스크립트를 이용한 에셋 추가

이 문서는 프로젝트 표준 자산 최적화 스크립트(`optimize:atlas`)를 사용하여 새로운 몬스터, 보스, 타일 이미지를 추가하는 방법을 설명합니다.

## 1. 이미지 준비 규격

이미지를 준비하실 때는 다음의 권장 해상도를 참고해 주세요.

| 종류 | 권장 해상도 (Px) | 파일 포맷 | 저장 경로 |
| :--- | :--- | :--- | :--- |
| **일반 몬스터** | 256x256 ~ 512x512 | PNG (투명) | `src/shared/assets/entities/` |
| **보스(Boss)** | 1024x1024 | PNG (투명) | `src/shared/assets/entities/` |
| **광물/타일** | 128x128 | PNG | `src/shared/assets/tiles/` |

> [!TIP]
> **투명 배경**: 게임 내 렌더링을 위해 반드시 배경이 없는 투명한 PNG로 작업해 주세요.

---

## 2. 자산 추가 및 아틀라스 업데이트 절차

수동으로 JSON을 수정할 필요가 없습니다. 다음 3단계만 거치면 됩니다.

### 단계 1: 이미지 파일 배치
준비한 이미지를 용도에 맞는 아래 경로에 넣습니다.
- 예: 새로운 몬스터라면 `src/shared/assets/entities/new_monster.png`

### 단계 2: 자동 패킹 명령어 실행
터미널에서 다음 명령어를 실행합니다.
```bash
npm run optimize:atlas
```
이 명령어는 모든 이미지를 모아 최적화된 WebP 아틀라스와 JSON 매니페스트를 자동으로 생성합니다.

### 단계 3: 코드에서 참조
`monsterData.ts` 등 설정 파일에서 해당 자산을 참조할 때는 **파일명(확장자 포함)**을 사용합니다.
```typescript
{
  id: 'new_monster',
  imagePath: 'new_monster.png', // 확장자까지 정확히 기재
  ...
}
```

---

## 3. 주의 사항
- **중복 이름 금지**: 서로 다른 폴더에 있더라도 파일명이 같으면 아틀라스 내에서 충돌이 발생할 수 있습니다. 고유한 이름을 사용해 주세요.
- **이미지 크기**: 아틀라스 최대 크기는 2048x2048입니다. 에셋이 이보다 많아지면 스크립트가 자동으로 여러 개의 아틀라스 파일(game-atlas-0, 1...)을 생성하여 관리합니다.
