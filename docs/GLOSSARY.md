# 📖 용어 사전 (Glossary)

Drilling RPG 프로젝트 내에서 사용되는 핵심 기술 용어 및 게임 도메인 개념을 정의합니다.

---

## 🛠️ 기술 용어 (Technical Terms)

### **ECS (Entity Component System)**
게임 개발에서 사용되는 데이터 중심(Data-oriented) 설계 패턴입니다.
- **Entity**: 고유 ID를 가진 단순한 컨테이너입니다.
- **Component**: 데이터만을 포함하는 구조체입니다. (예: `Position`, `Velocity`)
- **System**: 특정 핵심 로직을 처리하는 함수입니다. (예: `MovementSystem`은 `Position`과 `Velocity`를 가진 엔티티만 필터링하여 처리합니다.)

### **SoA (Structure of Arrays)**
데이터를 배열의 구조로 관리하는 방식입니다. (예: `[{x, y}, {x, y}]` 대신 `{x: [0, 1], y: [0, 1]}`)
- **CPU 캐시 효율**: 연속된 메모리 공간을 엑세스하므로 캐시 미스가 줄어들어 성능이 비약적으로 향상됩니다.
- **메모리 접근**: 동일한 컴포넌트 데이터만 읽어오므로 불필요한 데이터 로드를 방지합니다.

### **트리플 버퍼링 (Triple Buffering)**
렌더링 성능을 극대화하기 위해 3개의 버퍼를 교대로 사용하는 기법입니다.
- **Worker**: 현재 프레임의 데이터를 작성합니다. (Buffer 1)
- **Main**: 이전 프레임의 데이터를 읽어 보간(Lerp) 렌더링을 수행합니다. (Buffer 2)
- **Idle**: 다음 동기화를 위해 대기 중인 빈 버퍼입니다. (Buffer 3)
- 이 메커니즘을 통해 워커와 메인 스레드 간의 락(Lock) 없이 쾌적한 통신이 가능합니다.

### **FSD (Feature-Sliced Design)**
프론트엔드 프로젝트를 기능 단위로 쪼개어 관리하는 아키텍처입니다.
- **Layers**: `app`, `widgets`, `features`, `entities`, `shared` 순의 계층 구조를 가집니다.
- **고립성**: 하위 레이어는 상위 레이어를 참조할 수 없어 의존성 혼란을 방지합니다.

## 🎮 게임 도메인 용어 (Game Terms)

### **심연 (Abyss)**
게임의 주 무대가 되는 지하 세계입니다. 깊이(Depth)에 따라 자원의 가치와 몬스터의 난이도가 상승합니다.

### **룬 (Rune)**
드릴에 장착하여 고유한 스킬 효과나 능력치 버프를 부여하는 아이템입니다. 합성(Synthesis)을 통해 상위 등급으로 업그레이드할 수 있습니다.

### **드론 (Drone)**
플레이어를 따라다니며 채굴을 보조하거나, 자원을 제련(Smelting)하는 속도를 높여주는 지원 유닛입니다.

### **차원 이동 (Dimension Travel)**
현재 세계를 초기화하고 더 높은 난이도와 보상이 있는 다음 차원으로 넘어가는 시스템입니다. (프레스티지 시스템)
